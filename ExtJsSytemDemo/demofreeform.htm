    　关于自由表头
    
　　Supcan Treelist(硕正树列表)对数据的展现
方式是诸如Grid、List、Tree，可容纳多条记录，
而Supcan Freeform(硕正自由表头)则用于单条
记录、自由布局的场合。常见的基于 Master-
Detail展现方式的单证处理中，Detail部分通常是
Grid，而Master部分则是Freeform。

　　Supcan Freeform 中的输入控件大多数和
Supcan Treelist一致，实现了纯 HTML难以达到
的高效率输入。
　　Supcan Freeform也是根据 XML 描述文挡
动态创建的，其中的数据则独立于该 XML (也允
许混合)，需要通过 js 动态加载。

　　Supcan Freeform 也同样具备了强大的表达
式解析功能、事件触发机制，以及通过调用API 动
态更改属性、操纵数据的功能。

　　此外，Supcan Freeform 还提供了菜单、各
种风格的按钮、和背景及图片的渲染功能，大大
扩展了 Freeform 的用途，程序员甚至可以做出
丰富多彩的工具条。


    1.体验Freeform 体验和Treelist一样的输入控件、计算表达式、输入掩码、各种下拉，以及Enter、Tab、Shift+Tab键的全键盘操作、简练易懂的 XML 文档.
    2.多种掩码输入 演示掩码(EditMask)、输入范围，已内置了的合法验证可以省去不少js.
    3.多种下拉 和Treelist类似的各种下拉.
    4.关联下拉 和Treelist类似的关联下拉，通过事件、表达式动态更新下拉内容.
    5.事件、按钮、菜单、
    　工具条集中演示    		 如题，关注所有动作皆能抛出事件，包括输入、选菜单、击按钮。
    演示了格式按钮、菜单、工具条，均在 XML 创建文档中定义.
    6.动态更改属性(1) 可以通过js调用API，实现动态增删内容、替换内容、更改外观等等.
    7.动态更改属性(2) 一个结合TableLayout的更实用的例子：整块隐藏、展开功能.
    8.有表头表体的表单(1) Freeform 和 Treelist一起，分别作为表单的表头表体，实现了常见的类似 Master-Detail 的单据录入，并演示其数据的加载和提交方式.
    9.有表头表体的表单(2) 引入了Supcan Report组件，实现了单据的打印功能.
    10.多记录Freeform(1) Freeform 嵌进 Treelist中，作为Treelist的数据行来显示，扩展了Treelist的功能.
    11.多记录Freeform(2) 嵌入 Treelist 的Freeform采用TableLayout, 实现了Treelist不规则的表格的输入.
    12.多记录Freeform(3) Freeform 以树形式嵌进另一 Freeform中，以类似组织结构图形式展现，扩展了Freeform自身的功能.
    13.基于Freeform的对话框 Freeform的另一个重要用途：基于Freeform展现的对话框，实现了对话框百分之百的自定义.
    14.辅助Treelist输入(1) 通过事件和Treelist交互：Treelist当前行切换时刷新Freeform、Freeform修改内容后更新Treelist.
    15.辅助Treelist输入(2) Treelist使用内部freeform资源、以非模式对话框方式实现输入.

    16.Freeform设计器 基于TableLayout定位的在线的表单设计器， 用以辅助程序员设计Freeform表单.

    17.文件上传 这是Treelist没有的输入控件，专门原来实现文件上传，提供基于Http或Ftp的多种上传方案.
    18.炫酷ImageFlow
    　(图像流)控件    		 图片流控件，可用于产品图像的陈列浏览.
    19.选项卡 选项卡即Tab, 能包括多个页签，每个页签能容纳树、报表、树列表等大粒度组件.
    20.Freeform框架 这是今后将逐步完善的功能：Freeform作为总体布局的结构框架，能容纳树、报表、树列表这些大粒度组件.

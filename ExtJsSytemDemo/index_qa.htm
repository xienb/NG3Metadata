问1.硕正套件的学习版和正版有什么差别？
答：您注意到了在首次自动下载时有“...【演示版】”的提示了吗？正式版采用的将是完全不一样的
文字，此外，学习版在打印及预览时有“学习版，非商业用途”等提示以及行列数的一些限制。


问2.为什么demo包采用静态页面？在 JSP 或 ASP.Net中能使用吗？
答: 因为是纯客户端产品的关系，和后端(服务器端)技术的关系不那么密切，所以我们选择最通用的
静态页面作demo。不管是ASP还是JSP，HTML+CSS+JS是二者间最大的交集，换句话说，我们的产
品在静态页面上都能跑，更何况是动态页面。


问3.我在JSP或ASP.Net中使用硕正套件，为什么表格全是空的，好像是OnReady()没有执行？
答: 很有可能您把动态创建控件/插件的语句安置在<form> ... </form>内了，如果真是这样，您可以
参考一下开发文档的“使用指南 \ 9.Firefox和IE的一些差别”。
    如果还是不正常，建议您选一个demo页(例如第一个页面t1.htm)，将其修改成.jsp或.aspx文件，
这对您来说易如反掌吧？


问4.demo中的数据基本上都是XML文件，实际开发时也必须是XML文件吗？
答: 当然不是，应该是 Http Response 的数据包。在实际开发中，通常是由应用服务器从数据库中获
取数据，并将其封装成 XML(或JSON、TSV-TXT)，然后将这个XML返回给客户端，这个过程有点类似
于XML Web Service的对象封送，无论是 JavaEE 还是 ASP.Net，都有简单的数据序列化的实现方法。
　　在Java EE开发中，通常采用Servlet作为顶层来处理Http Request / Response的.


问5.demo页和文档从未提到数据库，硕正套件支持什么样的数据库？
答: 硕正套件不直接连接数据库，所以也不关心数据库。我们认为，访问数据库应该是应用服务器的事，
硕正套件只关心数据：基于Http request/response的数据。


问6.在IE下，硕正套件是一个ActiveX控件。我自己以前也开发过ActiveX控件，发现维护量很
大，有些客户根本就安装不了，硕正套件也会有这个问题吗？
答: 你自己开发的ActiveX控件八成是未经签名的，微软操作系统对未经签名控件的限制越来越严厉。
硕正套件ActiveX是经过了正规签名、实现了安全接口的，并且在运行过程中不写注册表、不往诸如
System32 等目录胡乱拷贝文件，所以在用户正常的IE安全等级下能够下载、运行，极少发现被操作系
统拦截、无法安装的情况.
